# app_settings keys

Ключи в таблице `app_settings`, которые использует приложение.

## Глобальные настройки

- `check_interval_minutes` — интервал авто-проверок в минутах (`0` = отключено).
- `last_checks_run_at` — отметка времени последнего фактического запуска `check.php`.
- `alert_email_recipients` — список email через запятую для уведомлений.

## Состояние уведомлений по сайту

Для сайта с id `N`:

- `site_state_N` — текущий подтвержденный статус (`unknown` / `up` / `down`).
- `site_fail_count_N` — количество подряд неуспешных проверок.
- `site_success_count_N` — количество подряд успешных проверок.

## Троттлинг старого DOWN-алерта

Старый ключ, может встречаться в БД от прежней логики:

- `last_down_email_site_N` — последнее время отправки DOWN-уведомления.

Примечание: новая логика алертов использует состояние и пороги антифлапа, поэтому этот ключ можно считать legacy.
